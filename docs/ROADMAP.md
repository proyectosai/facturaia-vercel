# Hoja de ruta

Este documento resume la dirección prevista para FacturaIA como producto y como proyecto open source.

## Prioridad actual

- pulir la experiencia de facturación diaria
- reforzar el estudio documental con mejores plantillas
- cerrar detalles de despliegue y hardening
- mejorar documentación para contribución externa

## Próximos bloques funcionales

### Facturación

- borradores persistentes en base de datos
- duplicación de facturas
- más estados visuales y acciones por lote
- exportaciones adicionales orientadas a asesoría

### Documentos

- plantillas más ricas para propuesta, contrato y presupuesto
- más opciones de branding y bloques reutilizables
- históricos y duplicación de documentos
- mejora de calidad de salida con IA local

### Suscripciones y producto

- onboarding más guiado
- mejor explicación de límites y capacidades por plan
- gating más fino en experiencia protegida
- mayor claridad entre demo y entorno real

### Fiscal y normativa

- más ayudas contextuales dentro del flujo
- revisión de copy legal y fiscal con referencias oficiales
- evolución futura hacia exportes compatibles con más escenarios reales

## Infraestructura y calidad

- más validaciones automáticas
- tests de rutas y utilidades críticas
- mejora de observabilidad
- pipeline de despliegue más formal

## Filosofía

FacturaIA quiere ser útil para autónomos españoles y, al mismo tiempo, servir como referencia práctica para aprender a construir producto real en abierto.
