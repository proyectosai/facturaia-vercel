# QA e ISO/IEC 25010

Fecha de revisión: 20 de marzo de 2026

## Alcance de esta revisión

Este documento resume la batería de comprobaciones repetibles que hoy ejecuta FacturaIA y su lectura práctica según las características de calidad de ISO/IEC 25010.

No es una certificación ISO formal.

Sí es una guía honesta para responder a estas preguntas:

- qué se está comprobando de verdad
- qué cubre la CI
- qué riesgos siguen abiertos
- qué áreas conviene reforzar antes de usar FacturaIA como pieza crítica

## Batería ejecutada

Comandos ejecutados en local:

```bash
npm run lint
npm run typecheck
npm test
npm run demo:build
npm run test:smoke
```

Cobertura actual de la batería:

- análisis estático con ESLint
- chequeo de tipos con TypeScript y tipos de rutas Next.js
- tests unitarios con Vitest
- build de producción en modo demo
- smoke test de rutas clave contra un `next start` real
- CI en Linux y Windows para detectar regresiones multiplataforma

## Lectura ISO/IEC 25010

### 1. Adecuación funcional

Estado: medio-alto

Fortalezas:

- el núcleo de facturación funciona y se puede recorrer de punta a punta
- hay continuidad operativa entre facturas, cobros, banca, CRM y documentos
- existen módulos fiscales y documentales claramente separados

Huecos:

- Facturae sigue en fase preparatoria, sin firma XAdES
- VeriFactu no está implementado como circuito de cumplimiento completo
- firma documental sirve como evidencia operativa, no como firma cualificada

### 2. Eficiencia del rendimiento

Estado: medio

Fortalezas:

- el build de producción es estable
- las páginas principales responden en demo

Huecos:

- no hay benchmarks formales
- no hay pruebas de carga para OCR, IMAP, banca o documentos IA
- la IA local depende totalmente del hardware del usuario

### 3. Compatibilidad

Estado: medio-alto

Fortalezas:

- se ha añadido CI en Linux y Windows
- se han corregido rutas frágiles por diferencias de entorno

Huecos:

- faltan pruebas reales en macOS en CI
- faltan pruebas de navegador end-to-end
- aún no hay matriz formal por versiones de Node o Supabase

### 4. Usabilidad

Estado: medio-alto

Fortalezas:

- mejor onboarding
- guía de 15 minutos
- asistente de primeros pasos
- documentación en español y madurez por módulo

Huecos:

- la instalación completa sigue siendo técnica para un autónomo medio
- faltan más ayudas in-app para Supabase, correo, IMAP y certificados

### 5. Fiabilidad

Estado: medio

Fortalezas:

- backups y restore existen
- smoke tests reducen roturas obvias en producción demo

Huecos:

- no hay test de restauración extremo a extremo
- no hay test real de recuperación tras fallo de base de datos
- no hay suite de regresión sobre migraciones

### 6. Seguridad

Estado: medio

Fortalezas:

- RLS y separación server-only
- cabeceras de seguridad
- validación estricta de uploads en puntos sensibles
- bloqueo si el documento firmado cambió

Huecos:

- no hay pentest externo
- no hay escaneo SAST/DAST adicional al lint
- falta endurecer mejor el manejo de certificados y secretos si se implementa firma fiscal real

### 7. Mantenibilidad

Estado: medio-alto

Fortalezas:

- TypeScript
- estructura modular
- documentación por módulo
- pruebas ya repetibles

Huecos:

- la superficie funcional ha crecido muy rápido
- faltan más tests de integración entre módulos

### 8. Portabilidad

Estado: medio-alto

Fortalezas:

- Docker
- demo en Vercel
- CI multiplataforma

Huecos:

- el modo real aún depende bastante de servicios y variables externas
- la instalación privada completa sigue necesitando una guía técnica

## Qué mejoraría primero

Prioridad 1:

- Playwright para flujos reales de facturación, cobros, firma y gastos
- tests de restore completo sobre base de datos temporal
- tests de integración para IMAP, SMTP y WebDAV con mocks

Prioridad 2:

- benchmarks de generación PDF, OCR y documentos IA
- validación de XML frente a esquemas XSD
- pruebas de firma fiscal con certificados reales de laboratorio

Prioridad 3:

- smoke fiscal dedicado para Facturae / VeriFactu
- matriz CI adicional con macOS
- trazabilidad automática de incidencias desde `/feedback`

## Conclusión

FacturaIA ya no está solo en “funciona en mi máquina”.

Con la batería actual, el proyecto verifica:

- calidad estática
- consistencia de tipos
- núcleo funcional mínimo
- build real de producción
- rutas principales en demo
- compatibilidad básica Linux/Windows en CI

Lo que todavía falta para entrar en una zona de confianza alta es:

- más integración real
- más fiscalidad verificable
- más pruebas de recuperación
- más automatización en firma y cumplimiento
